\section{Workflows}

\subsection{Comparable Evaluation Template}

For new experiments, prefer one parameterized command template over editing many
single-use scripts. The current repository already has this shape in
\texttt{explain.py}: it accepts seed, episode count, episode length, output
folder, submission path, reward mode, red-profile selection, and SHAP flags.

\begin{lstlisting}
python explain.py \
  --mode single \
  --strategy single \
  --single_profile fsm_default \
  --max-eps 100 \
  --episode-length 500 \
  --seed 42 \
  --phase_reward_mode default \
  --output Results/EVAL_NAME \
  --submission-path CybORG/Evaluation/submission
\end{lstlisting}

The fields that must be fixed before comparing two agents are:

\begin{itemize}
  \item \texttt{--seed}: controls scenario generation and stochastic choices.
  \item \texttt{--max-eps}: number of episodes.
  \item \texttt{--episode-length}: normally 500 for CC4.
  \item \texttt{--phase\_reward\_mode}: \texttt{default}, \texttt{contractor\_off}, or \texttt{red\_only}.
  \item \texttt{--single\_profile} or \texttt{--profile\_weights}: red attack distribution.
  \item \texttt{--submission-path}: exact agent implementation under test.
  \item \texttt{--output}: run-specific output directory; never reuse for another run.
\end{itemize}

For reward-decomposition experiments, set the output path and reward mode in the
same command and keep the generated JSONL reward log with the scalar summary.
For explainability experiments, add \texttt{--shap} plus fixed
\texttt{--shap-background-samples}, \texttt{--shap-explain-samples}, and
\texttt{--shap-random-state}.

\subsection{Heuristic Evaluation}

The repository README gives a direct evaluation pattern through the official
submission evaluator:

\begin{lstlisting}
python -c "
import sys; sys.path.insert(0, 'CybORG/Evaluation/submission')
from submission import Submission
from CybORG.Evaluation.evaluation import run_evaluation
run_evaluation(Submission, log_path='Results/v11a/', max_eps=100, seed=42)
"
\end{lstlisting}

For v11b, adjust the inserted submission path to \texttt{CybORG/Evaluation/submission\_v11b}.

For comparable fork experiments, the equivalent \texttt{explain.py} call is
usually preferable because it records action logs, reward decomposition, scalar
summaries, and optional SHAP artefacts in a single run directory:

\begin{lstlisting}
python explain.py \
  --mode single \
  --strategy single \
  --single_profile fsm_default \
  --max-eps 100 \
  --episode-length 500 \
  --seed 42 \
  --phase_reward_mode red_only \
  --output Results/v11b_red_only_fsm_default_seed42 \
  --submission-path CybORG/Evaluation/submission_v11b
\end{lstlisting}

\subsection{MARL Smoke Test}

The local workflow document recommends:

\begin{lstlisting}
source .venv/bin/activate
python marl/train.py mini_curriculum_smoke --mini \
  --strategy curriculum --warmup_updates 1 \
  --log_per_profile --device cpu
\end{lstlisting}

Expected outputs include reward logs under \texttt{logs/} and per-agent checkpoints under \texttt{checkpoints/}.

\subsection{Explainability Run}

The explainability pipeline can run a short single-profile pass:

\begin{lstlisting}
python explain.py \
  --mode single \
  --strategy single \
  --single_profile fsm_default \
  --max-eps 1 \
  --episode-length 5 \
  --non-heuristic \
  --shap \
  --seed 1337 \
  --phase_reward_mode red_only \
  --output SmokeResults/FULL_RUN \
  --submission-path .
\end{lstlisting}

Expected outputs include reward logs, scalar reward summaries, action logs, and SHAP analysis folders.
